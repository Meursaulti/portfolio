## 笔记

# C语言编译过程

写一个 simple.c，然后执行：

gcc simple.c -o simple

整个过程大致可以拆成下面几步：

---

## 1. 预处理（Preprocessing）

先做一轮“纯文本替换”：

- #include → 头文件直接展开
- #define → 宏替换
- #ifdef → 条件编译
- 注释会被删掉

为了避免头文件被重复 include，需要用 Header Guard：

#ifndef COMMON_H_
#define COMMON_H_

// 声明

#endif

这里要注意：

> 这一阶段完全不懂 C 语义，就是在做字符串替换

输出是 `.i` 文件

---

## 2. 编译（Compilation）

这一阶段才真正“看懂代码”：

- 语法 / 语义检查
- 生成中间表示（IR）
- 做一些优化
- 最后变成汇编（.s）

一个关键点：

> 函数可以没有实现，只要有声明就能过编译

比如：

call add

这里还是符号，不是地址

---

## 3. 汇编（Assembly）

把 `.s` 转成机器码，生成 `.o`

同时做三件事：

- 指令转二进制
- 建 symbol table
- 建 relocation table
  relocation table:  对于全局变量的读取或者存储和外部函数的调用,进行标记到链接阶段补全

## 4. 链接（Linking）

链接器生成可执行文件:

- 把多个 .o 拼起来
- 找函数实现
- 把之前没填的地址补上

分两种：

### 静态链接
从库直接拷需要的目标文件一起链接 → 文件变大但独立

### 动态链接（默认）
进行标识运行时加载库，节省磁盘空间，也可以使得库更新不影响编译好的代码
---

# 总结

.c  
→ 预处理 → `.i`  
→ 编译 → `.s`  
→ 汇编 → `.o`  
→ 链接 → 可执行文件  
→ 运行 → 内存加载 + 动态链接
